
JIGA BHAI BOT -- FINAL DEPLOYMENT GUIDE
=======================================

STEP 1: GITHUB SETUP
--------------------
1. Go to your GitHub repo: Jiga_Tradebot_automation
2. DELETE all old bot files:
   - jiga_bhai_railway_bot.py
   - jiga_bhai_FIXED_bot.py
   - jiga_bhai_HUMAN_MINDED_bot.py
   - jiga_bhai_REALTIME_bot.py

3. UPLOAD new file:
   - jiga_bhai_FINAL_bot.py (rename to jiga_bhai_railway_bot.py)
   OR keep name as jiga_bhai_FINAL_bot.py and update Procfile

4. Update requirements.txt:
   python-telegram-bot[job-queue]==20.7
   requests==2.31.0
   pytz==2024.1

5. Update Procfile (if needed):
   If file name is jiga_bhai_FINAL_bot.py:
   worker: python jiga_bhai_FINAL_bot.py

   If renamed to jiga_bhai_railway_bot.py:
   worker: python jiga_bhai_railway_bot.py

STEP 2: RAILWAY DEPLOYMENT
--------------------------
1. Go to railway.app dashboard
2. Find your project
3. Go to Deployments tab
4. Click "Redeploy" or trigger new deploy
5. Wait for build to complete

STEP 3: ENVIRONMENT VARIABLES
------------------------------
Go to Railway -> Variables tab and add:

REQUIRED:
- BOT_TOKEN = your_bot_token_from_BotFather
- CHANNEL_ID = -1001234567890 (your channel ID)
- ADMIN_IDS = 123456789 (your Telegram user ID)

OPTIONAL:
- VIP_CHANNEL_ID = -1009876543210
- TIMEZONE = Asia/Kolkata
- MIN_RRR = 3.0
- MAX_DAILY_CALLS = 4
- SKIP_LOW_QUALITY = true

STEP 4: VERIFY DEPLOYMENT
-------------------------
Check Deploy Logs for:
- "Bot running!" message
- No SyntaxError
- No "No such file" errors
- Config loaded successfully

STEP 5: TEST BOT
----------------
1. Message your bot on Telegram
2. Send /start command
3. Should get welcome message
4. Send /market command
5. Should get market data (or "Data fetch nahi ho raha" if market closed)

WHAT'S FIXED IN THIS VERSION:
=============================
1. All string literals are single-line (no multi-line " strings)
2. All messages built using list + join() method
3. No emojis in string literals that could cause encoding issues
4. Clean compile - verified with Python compile()
5. Simple, readable code structure

BOT FEATURES:
=============
- Real-time NSE data (3-sec polling)
- Quality control (score 60+ required)
- Time restricted: 9-11 AM and 1-3 PM only
- Morning compulsory call at 9:18 AM
- Human-like messages
- Smart risk management
- Max 4 calls per day
- Auto skip low-quality setups

TROUBLESHOOTING:
================
If "No such file" error:
  -> Check Procfile matches your file name
  -> Check file exists in GitHub repo

If "SyntaxError" error:
  -> Use ONLY the FINAL version
  -> Don't mix old and new files

If "Missing ENV VARS" error:
  -> Check all required variables are set
  -> BOT_TOKEN, CHANNEL_ID, ADMIN_IDS are mandatory

If bot crashes after running:
  -> Check Deploy Logs for specific error
  -> Make sure channel ID is correct
  -> Make sure bot is admin in channel
